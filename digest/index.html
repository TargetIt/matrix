三篇 SGEMM 原始资料，重新讲一遍

这个目录不是替代原文，而是给原文加一层中文解释。每篇都保留原始链接和技术脉络；新增内容统一放在绿色或橙色的标记块里，帮助读者从“矩阵乘法是什么”一路读到“为什么 CUTLASS 要分这么多层”。

阅读顺序

1先懂问题

SGEMM 就是在算很多行和列的点积。

2再懂慢点

慢通常不是乘法慢，而是数据搬得太贵。

3看优化阶梯

合并访存、SMEM、寄存器、向量化逐层上来。

4看工程实现

wangzyon 仓库把这些优化写成可对照的 kernel。

5看抽象框架

CUTLASS 把这些经验整理成可复用模板。

如果把 GPU 想成一个大厨房：Global Memory 是远处的大仓库，Shared Memory 是灶台旁边的小货架，Registers 是厨师手里的碗。高性能 SGEMM 的核心，就是别让厨师每炒一下菜都跑回仓库拿一次盐。
