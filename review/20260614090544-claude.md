## 二、残余问题

### 2.1 `cutlass-hierarchy` 仍无渐进式折叠（B3 部分残留）

`sgemm-practice` 和 `cuda-matmul-worklog` 都添加了 `<details>` 折叠面板，将二级内容默认隐藏。`cutlass-hierarchy` 仅增加了一段 `.guide` 文字引导，但所有面板（层级 SVG、当前 policy 指标、主循环流水）仍同时展示。

**影响**：Skill 要求 "Start with overview → let user click to drill into details"。cutlass-hierarchy 仍未满足。

**建议修复**：将"当前 policy"指标面板和"主循环流水"面板包裹在 `<details>` 中，默认只展示层级 SVG + 参数 slider，用户点击后再展开细节。

### 2.2 SVG 交互元素缺少显式 `tabindex`

CSS 已添加 `.node:focus`、`.point:focus`、`.bar:focus` 规则，但 JS 渲染的 SVG `<g class="node">` 和 `<circle class="point">` 没有 `tabindex="0"` 属性。无 tabindex 时这些元素不在 tab 导航序列中，键盘用户无法到达。

**影响**：Skill 的 "All controls reachable via keyboard" 未完全满足。

**建议修复**：在 JS 渲染 SVG 元素时添加 `tabindex="0"`，并在点击处理中增加 `keydown` 监听（Enter/Space 触发选中）。

### 2.3 `cutlass-hierarchy` 数据路径中 SMEM 占用计算与原文略有偏差

当前计算 `inputFloats = (bm + bn) * bk`，原文 CUTLASS 实际使用的公式为 `(bm * bk + bn * bk) * element_size`，这在数学上等价，但未计入输出 tile 的 SMEM 占用（CUTLASS 某些配置会将 C 子矩阵也缓存到 SMEM）。

**影响**：对理解 twin-buffer 场景下的精确 SMEM 限制有轻微误导。非致命。

**建议修复**：在当前政策旁添加注释说明 "此计算仅包含输入 A/B tile，未计入输出 C 的 SMEM 缓存"。
